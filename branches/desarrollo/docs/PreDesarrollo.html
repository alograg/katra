Waakun 3

Estructura de directorios

  • /

    • docs: Documentacion del desarrollo

      • PHP: Documentacion de PHP
      • MySQL: Documentacion de Base de Datos
      • JS: Documentacion de JavaScript
      • CSS: Documentacion de Cascading Style Sheets
      • APIs: Documentacion de Interfaz de Programación de Aplicaciones

    • lib: Contiene las librerias de PHP

      • js: Contiene librerias de JavaScript que se utilizan generalmente

      • css: Contiene librerias de CSS que se utilizan generalmente

      • php: Apartado para los archivos de PHP

        • externals: Contiele las librerias externas

          • AMFPHP
          • htmlparser
          • PHPMailer
          • reCaptcha
          • Smarty
          • Twitter

        • mekayotl: El framework de nuestros proyectos

          • database: Contenedor de los adaptadores a base de datos, abstracts e interface

            • mysql: Contiene los adaptadores para mysql
            • sqlite: Contiene los adaptadores para sqlite

          • tools: Contiene clases utilitarias

            • transfer: Utilerias para hacer transferencias de versiones anteriores a la actual

          • waakun: Ubicacion de la aplicacion Waakun 3

            • dal: Contenedor de los objetos de acceso a datos

              • access: Objetos de acceso a tablas

              • bussines: Objetos de logica de datos

              • value: Objetos de valor de datos

            • modules: Contiene los modulos que maneja Waakun 3

            • smarty: Contiene los plugins degenerados de Waakun 3 para smarty

            • aplications: Contiene look&feel, interacciones y servicios web base de la implementacion
              Ej: Waakun, Holbes, Quimera, etc.
              Tomando en cuenta que la implementacion base de llama Waakun, el resto de las implementaciones podran heredar archivos de esta
              Pot cada implementacion consiste en una carpeta con esta estructura:

              • js: Contiene los archivos de JavaScript especificos de la implementacion

              • languages: Contiene los archivos de lenguajes con los textos de la implementacion

              • style: Contiene los archivos del deiseño

                • extras: Archivos de CSS para custiones especiales (IE,print,mobile)

                • img: Contiene los archivos de imagenes utilizadas en el diseño

              • templates: Contiene directorios para cada formato de archivo solicitado

                • boletin: Distribucion html de para boletines

                • html: Distribucion html de para navegadores

                • rss: Distribucion html de para lectores de RSS

              • ws: Contiene archivos que suministran servicios especiales de la implementacion

    • web: Apartado para el contenido web

      • content: Carpeta de acceso publico, base del sitio

        • medias: Contenedora de archivos cargados

          • audios: Contiene solo archivos de audio (mp3,wav,aiff)

          • banners: Coniene solo archivos que son utilizados como banners (jpg,png,gif,swf)

          • docs: Coniene solo documentos (pdf,doc,xls,xml,ppt,txt,xslt,html)

          • files: Coniene solo archivos que no estan categorisados (zip,bin,tar,rar,*)

          • images: Coniene solo archivos de imagen (jpg,gif,png)

            • m: Contiene imagenes manipuladas del directorio padre

          • videos: Coniene solo archivos de video (flv,3gp,avi,mov,wmv,mp4,f4v)

        • models: Contiene look&feel, interacciones y servicios web exlusivos
          Pot cada modelo consiste en una carpeta con esta estructura:

          • js: Contiene los archivos de JavaScript especificos del modelo

          • languages: Contiene los archivos de lenguajes con los textos del modelo

          • style: Contiene los archivos del deiseño

            • extras: Archivos de CSS para custiones especiales (IE,print,mobile)

            • img: Contiene los archivos de imagenes utilizadas en el diseño

          • templates: Contiene directorios para cada formato de archivo solicitado

            • boletin: Distribucion html de para boletines

            • html: Distribucion html de para navegadores

            • rss: Distribucion html de para lectores de RSS

          • ws: Contiene archivos que suministran servicios especiales al modelo
